3 approches pour l'app Windows

Avec trade-offs — ma recommandation est marquée ⭐

A

⭐ PyInstaller + CustomTkinter

Stack : Python pur — CustomTkinter pour l'UI, PyInstaller pour packager en .exe, Inno Setup pour l'installateur Windows

✅ Avantages
Zéro réécriture du scraper
Tu connais Python
UI moderne et propre
1 seul langage
⚠ Inconvénients
.exe assez lourd (~80 MB)
Moins flexible qu'un web UI
⏱ ~2-3 semaines 📦 .exe ~80 MB 🎯 Difficulté : Facile
B

Electron + FastAPI

Stack : Electron (Chromium + Node.js) pour l'interface web, FastAPI (Python) comme backend local, communication via HTTP localhost

✅ Avantages
UI ultra-polished (HTML/CSS)
Tu connais le web
Code UI réutilisable → app web
⚠ Inconvénients
.exe très lourd (~250 MB)
2 processus à gérer
Architecture complexe
Node + Python bundlés
⏱ ~4-5 semaines 📦 .exe ~250 MB 🎯 Difficulté : Moyenne
C

Tauri + FastAPI

Stack : Tauri (Rust + WebView2 natif Windows) pour la fenêtre, FastAPI (Python) comme backend local — UI en HTML/CSS/JS

✅ Avantages
Très léger (~10 MB)
UI web (HTML/CSS)
Natif Windows
⚠ Inconvénients
Requiert Rust pour build
Plus complexe à setup
Python bundle séparé
⏱ ~5-6 semaines 📦 .exe ~15 MB 🎯 Difficulté : Difficile
💡 Ma recommandation : A (PyInstaller + CustomTkinter)
Tu gardes tout ton code Python existant, l'UI CustomTkinter peut être très moderne et propre, et la complexité reste raisonnable. Electron est tentant vu tes skills web, mais l'architecture à 2 processus (Node + Python) complique beaucoup les choses pour un gain d'UI qui ne justifie pas le temps investi.
