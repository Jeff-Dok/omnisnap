Architecture — Web Scraper Windows

Structure des fichiers et flux de données

Structure du projet
scraper-app/
main.py ← point d'entrée — lance l'app
gui/ ← tout ce qui est interface
app.py ← fenêtre principale CTk
sidebar.py ← navigation gauche
panels/
scrape_panel.py ← formulaire URL + modes
log_panel.py ← log temps réel
preview_panel.py ← aperçu résultats
history_panel.py ← scrapes précédents
scraper_modules/ ← TON CODE EXISTANT — inchangé ✓
build/
scraper.spec ← config PyInstaller
installer.iss ← script Inno Setup
history.json ← historique local (AppData)
Flux quand l'utilisateur lance un scrape
🖱 UI Thread
Clic "Lancer"
→
⚙ Worker Thread
scraper_modules (ton code)
→
📬 Queue
messages de log
→
🖥 log_panel
affiche en temps réel
ℹ Le scraping tourne dans un thread séparé pour que l'interface reste réactive. Une Queue Python (thread-safe) transmet les messages de log au panel sans risque de conflit.
Packaging
1. PyInstaller
Crée un dossier dist/scraper/ avec le .exe + toutes les dépendances Python bundlées
2. Inno Setup
Prend le dossier dist/ et génère Scraper_Setup.exe — installateur avec raccourcis, menu Démarrer, désinstallation propre
