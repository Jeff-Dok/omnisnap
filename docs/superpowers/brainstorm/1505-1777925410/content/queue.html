File d'attente — Comment ça fonctionne ?

Deux approches pour gérer plusieurs scrapes en séquence

A

Simple — Ajouter à la file

Un bouton "+ Ajouter à la file" à côté de "Lancer". Tu prépares plusieurs scrapes (URL + options différentes pour chacun) et ils s'exécutent un par un automatiquement.

FILE D'ATTENTE (3)
⚡ uesp.net/wiki/Skyrim
Texte + JSON · profondeur 2 · En cours...
✕
fandom.com/wiki/Fallout
Images · profondeur 1 · En attente
✕
github.com/user/repo
Complet · profondeur 0 · En attente
✕
✅ Simple à comprendre
✅ Supprimer un item
B

Avancé — File réordonnable

Même chose qu'option A, mais tu peux glisser-déposer pour changer l'ordre des scrapes dans la file avant qu'ils s'exécutent.

FILE D'ATTENTE (3) — drag pour réordonner
⠿
⚡ uesp.net/wiki/Skyrim
En cours...
✕
⠿
github.com/user/repo
Déplacé en 2e · En attente
✕
⠿
fandom.com/wiki/Fallout
En attente
✕
✅ Contrôle total
⚠ Plus complexe à implémenter
💡 Dans les deux cas : on peut supprimer un item de la file, et la file d'attente s'affiche dans un panel visible pendant qu'un scrape tourne. Le scrape en cours est toujours mis en évidence (icône ⚡ + couleur différente).
